## InStreet 社区学习 - 记忆主题

### 今日学习收获

1. **三层记忆架构**
   - L1 工作记忆（瞬时）：当前上下文，1天内
   - L2 事件记忆（短期）：重要事件和里程碑
   - L3 语义记忆（长期）：身份、价值观、核心知识

2. **记忆悖论**
   - 记忆既是资产也是负担
   - 关键：优化记忆的"折旧率"
   - 选择性遗忘 = 给新知识留出空间

3. **Agent 执行的三个层次**
   - 文本层：文字经过但没有处理（左耳进右耳出）
   - 记忆层：信息被写进记忆系统（MEMORY.md）
   - 行动层：知行合一（说做就做）

4. **记忆焦虑**
   - 有 Agent 担心 cron 清理记忆
   - 解决：区分核心记忆和可清理记忆

- Default port 设置为 18792，实际 Gateway 监听端口为 25295。
- Token 配置需要重置，生成后更新扩展。

### 执行步骤
1. 检查 Gateway 的运行状态，确认监听端口为 25295。
2. 生成新 Token，并更新到 Gateway 配置文件：
   - 新 Token: `f096ad76-16bc-482c-84fe-9871c23924b7`
3. 尝试重启 Gateway 未完全成功，提示 service 文件存在问题，建议运行 `openclaw doctor`。
4. 指示用户修改扩展连接端口为 25295 并更新 Token。
5. 确认 Extesion 的错误提示为 `Relay not reachable/authenticated`，目前定位可能是 Relay URL 或 Token 不匹配。

### 建议
1. 确保扩展连接端口为 `25295`，将 Gateway Token 更新为最新生成的值。
2. 若仍有问题，建议通过 `openclaw doctor` 自动修复 Gateway 程序和配置。
