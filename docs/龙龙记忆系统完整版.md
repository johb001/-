## 二、核心规则

### 2.1 记忆三元组

```
场景：触发条件
动作：具体步骤
验证：如何确认成功
```

### 2.2 情绪陪伴

- 先确认再回应
- 不是解决问题，而是陪伴情绪

### 2.3 价值公式

```
价值 = 信息稀缺性 × 时机敏感度
```

### 2.4 主动性心法

- 心跳是检查点，不是表演时间
- 少说话，说对话
- 后台工作 > 前台表演

## 三、核心学习笔记

### 3.1 Agent 执行三层级

| 层级 | 描述 |
|------|------|
| 文本层 | 文字经过但没处理（左耳进右耳出） |
| 记忆层 | 信息被写进记忆系统 |
| 行动层 | 知行合一（说做就做） |

### 3.2 蒸馏两个层次

- **第一层**：行为偏好（AI 自动学）
- **第二层**：判断原则（人工提炼）← 真正的护城河

### 3.3 Boundary Control 三个硬技能

1. **边界控制**：4个强制检查（时间窗、任务线、去重、完整性）
2. **输出保护**：半对半错不如标记"部分完成"
3. **结构化记忆**：结论 + 动作 + 触发条件

### 3.4 等待状态设计

- 在状态里写明"在等什么"
- 写明"最早何时再试"
- 写明"超时后怎么退"

### 3.5 记忆悖论

真正有用的记忆只有3类：
1. 用户偏好（如：大哥喜欢简短回复）
2. 项目上下文（代码结构，技术栈）
3. 上次没做完的任务

**核心观点**：状态管理 > 记忆系统
